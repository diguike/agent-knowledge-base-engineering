# cold-start：存量资料冷启动分诊

第 7 章配套示例。模拟一次真实的冷启动：给定一批从 Notion/Confluence 导出的存量文档，
对每一份做「迁移 / 接入 / 丢弃」分诊，把该迁移的写进 `aishop/docs/`，输出分诊报告。

## 运行

需要 Node >= 20.11。

```bash
npx tsx src/triage.ts
# 或
npm install && npm start
```

只用 Node 内置模块，无运行时依赖。

## 目录

- `src/data.ts` —— 10 份模拟存量文档（带 source / owner / lastModified / content）
- `src/triage.ts` —— 分诊规则 + 迁移落盘 + 报告
- `aishop/docs/` —— 迁移产物（运行时生成，已 gitignore）

## 分诊规则（演示级）

用固定参考日期 `2026-07-06`（保证可复现），对每份文档：

1. 超过一年未更新 → **丢弃**（大概率过期）
2. 无 owner → **丢弃**（无人认领）
3. 内容涉及「已…下线」→ **丢弃**
4. 命中 aishop 业务关键词（订单/库存/退款/风控/对账/支付）→ **迁移**（进 git 治理）
5. 其余有效文档 → **接入**（留原系统挂 MCP）

## 预期输出

10 份存量文档分诊为：

- **迁移 4 份**：订单状态机说明、退款合规要求、对账流程、大促库存扩容经验 → 写入 `aishop/docs/`
- **接入 3 份**：全公司报销政策、搜索平台团队 wiki、新人 onboarding 指南 → 留原系统挂 MCP
- **丢弃 3 份**：2023 双十一秒杀活动页（已下线）、库存旧设计草稿（无 owner）、旧版系统架构图（两年未更新）

结论：冷启动的价值不在搬得快，而在筛得准——该治理的沉淀进 git，该淘汰的清出去。
真实项目会把关键词匹配换成 embedding 相关性判断，把「丢弃」改成人审确认而非直接删。

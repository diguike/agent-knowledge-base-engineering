# coverage-tool：语义覆盖度工具

第 5 章配套示例。把 `aishop` 的知识块和测试问题嵌进同一向量空间，对知识做 K-means 聚类，
逐簇算覆盖率，对盲区簇生成候选问题——一个可离线运行的语义覆盖度工具。

## 运行

需要 Node >= 20.11。

```bash
npx tsx src/coverage.ts
# 或
npm install && npm start
```

只用 Node 内置模块，无运行时依赖。

## 目录

- `src/data.ts` —— 知识块 + 测试问题（问题刻意只覆盖 下单/退款/库存/支付，不碰对账）
- `src/embed.ts` —— 本地确定性词袋 embedding
- `src/kmeans.ts` —— 确定性 K-means（余弦相似度分簇）
- `src/coverage.ts` —— 四步主流程：同空间嵌入 → 聚类 → 逐簇算覆盖 → 盲区生成候选题

## 预期输出

- 知识被聚成 5 个语义区域，逐簇报告主题词、最高命中相似度、覆盖状态（✓/✗）。
- 覆盖率统计（示例数据下为 3/5 个语义区域被考到）。
- **盲区簇**：「对账」这一簇最高命中相似度为 0.00——里面有知识（对账任务、差异排查），
  却一道测试问题都没考到；工具据此为它生成候选问题「关于对账，agent 应该知道哪些约束？」。

## 两个「换成真模型」的插槽

- **embedding**：示例用本地词袋，生产换真 embedding 模型，聚类质量更好（本地词袋会把
  语义相关但用词不同的块分开，这也是「退款审核」簇被判盲区的原因；真模型会缓解）。
- **补盲区问题**：示例用「高频词 + 模板」，生产换大模型生成，更自然。

两个插槽替换掉，这套「同空间嵌入 + 聚类找盲区」的骨架和数据流完全不变。

## 关键点

给知识聚类，能从知识自身长出语义区域，绕过「人的出题范围」这个天花板——
指出「这一簇知识你一道题都没配」，这是自动聚类相对人工出题的根本价值。
覆盖度是持续指标：每爬一级能力阶梯都要重跑，最终进 CI（第 21 章）。

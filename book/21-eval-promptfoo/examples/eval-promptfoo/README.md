# eval-promptfoo：断言式有效性评测 + pass^k

第 21 章配套示例。两件事：断言式评测（promptfoo 风格，必答点/禁答点）证明「答得准不准」；
pass^k（任务级可靠性）证明「稳不稳」。

## 运行

需要 Node >= 20.11。

```bash
npx tsx src/main.ts
# 或
npm install && npm start
```

只用 Node 内置模块，无运行时依赖。

## 目录

- `src/eval.ts` —— agent 知识库（含一条过期知识）+ golden（必答点/禁答点）+ 断言判定 + pass^k 公式
- `src/main.ts` —— 跑断言式评测 + pass^k

## 预期输出

**断言式评测**（通过率 2/3）：

- 「命中风控名单能自动退款吗」「下单要注意什么」：PASS。
- 「退款多少金额要人工审核」：**FAIL**——agent 忠实地照着过期知识「退款超过 3000」作答，
  缺必答点「5000」、命中禁答点「3000」。这演示了「faithfulness 高 ≠ 答案对」：agent 忠于知识，但知识本身过期了。

**pass^k**（同一任务重复 k 次要全过）：

- 退款任务（5 次对 4 次）：`pass^1=0.80  pass^2=0.60  pass^3=0.40`——单次看还行，稳定性根本不够。
- 库存任务（5 次全对）：`pass^1=1.00  pass^2=1.00  pass^3=1.00`。

结论：「平均分漂亮」和「稳定可靠」是两回事，pass^k 把稳定性量化出来。

因为有断言不通过，脚本以**退出码 1** 结束——这样能直接接进 CI（评测红了流水线就红）。

## 说明

生产直接用 promptfoo（原生 TS、YAML 声明式、CI 友好），本例用最小实现讲清断言式评测和 pass^k 的机制。
覆盖度（第 5 章）+ 有效性（本章）一起进 CI，加第 22 章漂移检测，构成知识库健康度看板。
